In the next few minutes you will scaffold a report bundle, preview it live in your browser, and export a PDF — the core loop you will use every day with bino. The scaffolded project includes sample CSV data, SQL queries, charts, and tables so you can see how the pieces fit together before building your own.

This guide assumes you have `bino` installed and available on your `PATH`.

<Steps>

1.  **Create a new workdir**

    A **workdir** is the root directory of a report bundle.
    Run `bino init` to scaffold a new project. By default, it creates a directory named `rainbow-report`.

    ```bash
    bino init
    ```

    Without flags, `bino init` asks a few questions (target directory, report name, title, language).
    Use `-y` / `--yes` to accept defaults non-interactively.

    <Aside>
      You can specify a custom directory name using the `--directory` flag:
      ```bash bino init --directory my-report ```
    </Aside>

2.  **Inspect the generated files**

    Navigate into the newly created directory:

    ```bash
    cd rainbow-report
    ```

    After initialization, the workdir contains YAML manifests, data files, and helper files.
    Typical contents include:

    - A `bino.toml` project configuration file marking the project root
    - One or more `ReportArtefact` manifests describing the report outputs
    - `DataSource` and `DataSet` manifests defining input data and SQL transforms
    - At least one `LayoutPage` describing page layout
    - `ComponentStyle` and `Internationalization` manifests for styling and translations
    - A `.bnignore` file tuned for bino caches and build artefacts

    Use your editor to explore these files.
    The manifests all share `apiVersion: bino.bi/v1alpha1` and a `kind` that selects the spec.
    Each kind has a specific role: `DataSource` loads raw data, `DataSet` runs SQL on top of it, `LayoutPage` defines the visual layout, and `ReportArtefact` assembles pages into a PDF. See [Key ideas](/getting-started/key-ideas/) for the full mental model.

3.  **Preview the report**

    Start a live preview server from the workdir:

    ```bash
    bino preview
    ```

    bino scans the directory for manifests, starts a local HTTP server (by default `http://127.0.0.1:45678/`), and opens your default browser.
    As you edit YAML or data files, it reloads the preview automatically.

    <Aside type="tip">
      Run `bino preview --log-sql` to see the SQL queries used to populate
      datasets.
    </Aside>

4.  **Build a PDF**

    When you are happy with the preview, export a PDF:

    ```bash
    bino build
    ```

    This runs schema validation, executes datasets in the embedded query engine, renders HTML for each report artefact, and converts the pages to PDF using the rendering engine.
    By default, PDFs are written into a `dist/` directory under the workdir.

    Example using a custom output directory:

    ```bash
    bino build --out-dir dist/reports
    ```

</Steps>

## What you just built

You created a complete report bundle: data sources feed into SQL queries, queries power charts and tables, and a layout page composes them into a PDF. This is the [Reporting as Code](/why/reporting-as-code/) workflow — everything is a text file, everything is reproducible, and the same `bino build` command works on your laptop and in CI.

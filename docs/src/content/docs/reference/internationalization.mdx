---
title: Internationalization
description: Manage translations for report text by language and namespace.
---

`Internationalization` manifests hold localized messages for one language and optional namespace.
Layouts and components refer to keys; the renderer selects messages based on `ReportArtefact.language` and namespaces.

## Spec outline

```yaml
apiVersion: bino.bi/v1alpha1
kind: Internationalization
metadata:
  name: systemTexts_de
spec:
  code: de-DE
  namespace: _system
  content: {} # object or JSON string
```

Fields:

- `spec.code` ‚Äì locale code, for example `de-DE` or `en-US`.
- `spec.namespace` ‚Äì optional namespace to group keys.
- `spec.content` ‚Äì key/value map as object or JSON string.

## YAML object content

```yaml
---
apiVersion: bino.bi/v1alpha1
kind: Internationalization
metadata:
  name: systemTexts_de
spec:
  code: de-DE
  namespace: _system
  content:
    report.title.sales_overview: "Sales Overview"
    report.subtitle.q1_2024: "Q1 2024"
    card.title.revenue: "Revenue"
    card.title.ebit: "EBIT"
```

## JSON string content

```yaml
---
apiVersion: bino.bi/v1alpha1
kind: Internationalization
metadata:
  name: productTexts_en
spec:
  code: en-US
  namespace: products
  content: >-
    {"product.applications":"Applications","product.infrastructure":"Infrastructure"}
```

You can organize translations by language, business domain, or report, as long as namespaces and keys stay consistent with layout conventions.

## Attribute Reference

### Common Metadata

| Attribute | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `apiVersion` | string | yes | ‚Äî | Must be `bino.bi/v1alpha1`. |
| `kind` | string | yes | ‚Äî | Must be `Internationalization`. |
| `metadata.name` | string | yes | ‚Äî | Unique identifier for this translation set. |
| `metadata.labels` | object | no | ‚Äî | Key-value pairs for categorization and constraint matching. |
| `metadata.annotations` | object | no | ‚Äî | Arbitrary key-value metadata, not used by the system. |
| `metadata.description` | string | no | ‚Äî | Free-form description. |
| `metadata.constraints` | array | no | ‚Äî | Conditional inclusion rules. See [Constraints](/concepts/constraints-and-scoped-names/). |

### Spec Attributes

| Attribute | Type | Required | Default | Description | Sample |
|-----------|------|----------|---------|-------------|--------|
| `spec.code` | string | yes | ‚Äî | Locale code (BCP 47 tag). | `code: de-DE` |
| `spec.namespace` | string | no | ‚Äî | Optional lookup namespace for the translation block. Components reference translations by namespace. | `namespace: _system` |
| `spec.content` | object or string | yes | ‚Äî | Key-value map of translations. Can be a YAML object or a JSON string. | see below |

#### Content as YAML object

```yaml
spec:
  content:
    report.title: "Sales Overview"
    card.title.revenue: "Revenue"
    card.title.ebit: "EBIT"
```

#### Content as JSON string

```yaml
spec:
  content: >-
    {"report.title":"Sales Overview","card.title.revenue":"Revenue"}
```
